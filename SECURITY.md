# Security Policy

## Supported Versions

The following versions of F1 PUTwall are currently supported with security updates:

| Version              | Supported          |
| -------------------- | ------------------ |
| Latest (Main branch) | :white_check_mark: |
| All older versions   | :x:                |

Only the most recent release on the `main` branch receives security patches.

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public issue** for security-related matters. Opening a public issue exposes the vulnerability to malicious actors before a patch can be developed.

Instead, please report vulnerabilities privately using GitHub's native **Private Vulnerability Reporting** feature:

1. Navigate to the main page of this repository on GitHub.
2. Click on the **Security** tab located under the repository name.
3. In the left sidebar, click on **Vulnerabilities**.
4. Click **Report a vulnerability** to submit your findings securely to the maintainers.

When reporting, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue (including a Proof of Concept if available)
- The potential impact and risk level
- Any suggested mitigations or fixes
- Your environment details (OS, Python version, dependencies, etc.)

## Response Process

1. **Acknowledgment**: We will acknowledge receipt of your private security report within 5 business days.
2. **Investigation**: We will investigate, validate, and triage the vulnerability.
3. **Resolution**: We will work to develop, test, and apply a fix.
4. **Disclosure**: Once the fix is merged and released, we will coordinate a public security advisory. For critical vulnerabilities, we aim to resolve the issue within 30 days of acknowledgment.

## Scope

This security policy applies to the codebase, dependencies, and infrastructure directly managed within this repository.
